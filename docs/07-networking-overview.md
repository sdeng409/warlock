# 07. Networking Overview

## 1) 목표
- 최대 8인 실시간 PvP를 MVP 범위에서 안정적으로 완주

## 2) MVP 네트워크 원칙
- Dedicated Server는 MVP에서 제외 (Production 단계에서 재검토)
- Host/Listen + Relay 기반으로 시작 (**확정**)
- Matchmaking은 고급 자동매칭 대신 "방 생성 + 코드 초대" 우선 (**확정**)

## 2-1) 권한 모델 결정(확정)
- MVP: **Host + Relay** (친구 테스트/빠른 검증 우선)
- Dedicated: 당장 미도입, **서비스가 production 단계로 확장될 때 재검토**

## 2-2) 매치메이킹 정책(확정)
- MVP: **방 생성 + 초대 코드**
- 랜덤 매칭: 후속 버전(v0.2+)에서 검토

## 2-3) 룸 설정 정책(확정)
- 라운드 수는 방 생성 시 호스트가 선택(3/5/7)
- 기본값은 5라운드
- 최종 승자는 누적 순위 점수 기준으로 판정
- 동점자 처리: 라운드 1위 횟수 -> 마지막 라운드 순위 순으로 판정

## 3) 동기화 대상(최소)
1. 플레이어 위치/방향
2. 스킬 발동 이벤트(매핑 슬롯: QWER/ASDF/ZXCV)
3. 피해량/체력
4. 상태이상(슬로우/스턴 등)
5. 경기 상태(라운드 시작/종료/다음 라운드)
6. 순위 보상 골드/잔액
7. 상점 구매 트랜잭션(구매 성공/실패)
8. 라운드 사이 슬롯 재배치 결과
9. 라운드 수 설정(3/5/7)
10. 누적 순위 점수/최종 우승 판정
11. 아레나 축소 상태(현재 반경/최소 반경 도달 여부)


## 3-1) 배틀로얄 규칙 반영(확정)
- 승리 조건: 마지막 1인 생존
- 리스폰: 없음(사망 시 관전 전환)
- 사망 처리: 리스폰 타이머 대신 관전 전환 이벤트를 동기화
- 경계 DoT 수치: 유예 없음, 1초마다 최대체력 5% 피해
- 복귀 처리: 경계 안으로 복귀 시 DoT 즉시 중단
- 악용 방지: 경계 밖 누적 타이머를 유지하고, 복귀 즉시 0으로 리셋하지 않음
- 리셋 조건: 경계 안에 **연속 3초 이상** 머물렀을 때만 누적 타이머 초기화

## 3-2) 라운드 경제 규칙 반영(확정)
- 라운드 종료 후 순위 기반 골드 지급
- 골드는 라운드 사이 상점에서 스킬 구매에 사용
- 자원 시스템은 Cooldown only(마나/에너지 없음)

## 3-3) 아레나 축소 규칙 반영(확정)
- 축소 시작: 라운드 시작 30초 후
- 축소 속도: 초당 반경 1.5% 감소
- 최소 반경: 초기 반경의 35%
- 최소 반경 도달 후 추가 축소 중지

## 4) 품질 목표(초안)
- 일반 환경(약 80~120ms)에서 조작 가능 수준 유지
- 매치 중 강제 종료/동기화 붕괴 빈도 최소화

## 5) 단계적 확장
- 0.1: 커스텀 룸 + 8인
- 0.2: 빠른 입장(Quick Join)
- 0.3+: 필요 시 Dedicated/권한 강화 검토


## 6) 모드 확장 대비 설계 메모
- MVP(v0.1): 개인전(FFA) 규칙만 구현
- 후속(v0.2): 2v2 팀전 추가 예정
- 확장 포인트: 플레이어 상태에 `teamId` 필드를 미리 고려해 데이터 구조 변경 비용을 낮춤
