# 00. One Pager

작성일: 2026-03-04
프로젝트명: **warlock**

## 1) 게임 한 줄 정의
최대 8인이 좁은 탑뷰 콜로세움에서 자유 매핑 스킬 키셋(QWER/ASDF/ZXCV)으로 싸우는 실시간 PvP 아레나 게임.

## 2) 핵심 재미(Core Pillars)
1. 짧고 밀도 높은 교전
2. 스킬 연계/카운터 플레이
3. 넉백 + 경계 위험 + 아레나 축소로 전장 압축

## 3) 0.1 Vertical Slice 범위
- 플레이어: 2~8인(커스텀 룸)
- 입력: **우클릭 이동 + 좌클릭 대상/방향 지정 + 스킬 키(QWER/ASDF/ZXCV)**
- 맵: 1개(원형 콜로세움)
- 캐릭터: **1개(공용 베이스 캐릭터, MVP 고정)**
- 스킬: **클래스리스 스킬 풀 + QWER/ASDF/ZXCV 최대 12슬롯 자유 매핑**
- 라운드 승패: Last man standing (각 라운드 마지막 1인 생존)
- 리스폰: **없음(사망 시 관전 전환)**
- 경계 DoT: **유예 없음, 1초마다 최대체력 5% 피해**
- DoT 악용 방지: **경계 밖 누적 타이머(복귀 즉시 리셋 금지)**
- 아레나 축소 정책: **시간 경과에 따라 점진 축소 + 최소 크기 보장**
- 아레나 축소 수치: **시작 30초 / 초당 반경 1.5% 감소 / 최소 반경 35%**
- 난이도 목표: **입문자 친화(캐주얼 우선)**
- 모드: **v0.1 개인전(FFA) only**
- 확장 계획: **v0.2에서 2v2 팀전 추가**
- 로드아웃 목표: 동일 캐릭터라도 스킬 조합으로 운용 차별화
- 배포 목표: **Windows 테스트 빌드 only (MVP)**
- 네트워크 모델: **MVP는 Host + Relay, Dedicated는 후속(Production) 검토**
- 매치메이킹: **MVP는 방 생성 + 초대 코드**, 랜덤 매칭은 후속
- 경기 구조: **멀티 라운드 + 라운드 종료 후 골드 지급 + 스킬 상점 구매**
- 라운드 수: **방 생성 시 호스트 설정(3/5/7, 기본값 5)**
- 최종 우승 판정: **누적 순위 점수 최고 플레이어**
- 동점자 처리: **라운드 1위 횟수 > 마지막 라운드 순위** 우선
- 상점 정책(MVP): **고정 스킬 카탈로그, 리롤/재판매 없음**
- 레거시 감성 목표: **Warlock류(라운드+상점+밀어내기) 전투 템포 재현**
- 레거시 시스템 목표: **넉백 중심 장외 처치 + 아레나 축소 압박**

## 4) 제외 범위(0.1에서 미포함)
- 랭크/MMR/시즌
- 복잡한 매치메이킹
- 관전/리플레이
- 스킨/상점

## 5) 성공 기준
- 8인 방 생성/입장/매치 종료가 끊기지 않고 동작
- 기본 이동 + 12키 스킬 매핑 + 경계 DoT 규칙이 일관되게 작동
- 10회 내부 테스트 중 치명적 진행 불가 버그 0건
- 최소 2개 이상의 유효 로드아웃(플레이 감각이 다름) 검증
- 라운드 종료 후 골드 지급과 상점 구매가 끊김 없이 순환

## 6) Inspiration & IP Guardrail
- 영감 원천: **Warcraft III: Frozen Throne 커스텀 유즈맵 경험에서 영감**
- 가드레일: 원작의 고유 캐릭터/명칭/스토리/아트 리소스는 직접 사용하지 않고, 전투 구조 아이디어만 재해석
