# 05. Skill System (Classless Loadout)

작성일: 2026-03-04

## 1) 핵심 방향 (확정)
- 설계 기준: 원작 감성 유지를 위해 넉백 계열 스킬을 핵심 축으로 둔다.
- **캐릭터 고정, 스킬 자유 조합**: 캐릭터별 고정 스킬 구조를 사용하지 않는다.
- 플레이어는 **스킬 풀**에서 원하는 스킬을 선택해 원하는 키에 배치한다.
- 키 범위는 고정: **Q W E R / A S D F / Z X C V (최대 12 슬롯)**

## 2) 입력/슬롯 규칙
- 슬롯 키 집합: `QWERASDFZXCV`
- 각 슬롯에는 스킬 1개만 배치 가능
- 동일 스킬을 여러 슬롯에 중복 배치하는 것은 MVP에서 금지(밸런스/UX 단순화)
- 매치 시작 전 로드아웃 확정, 매치 중 키 재배치는 MVP에서 비활성

## 3) 클래스리스 로드아웃 구조
- 모든 플레이어는 동일한 베이스 캐릭터 사용
- 전투 스타일 차이는 **로드아웃 조합 + 키 배치 습관**에서 발생
- 목표: 캐릭터 제작 비용을 줄이고, 스킬 설계/밸런스에 집중

## 4) 스킬 데이터 공통 스키마(문서 기준)
- SkillId
- Name
- Tags (Damage / Mobility / Control / Defense / Utility)
- CastType (Self / Target / Direction / Area)
- CooldownSec
- CastTimeSec
- Range
- Radius (필요 시)
- Damage
- CCType / CCTimeSec
- ResourceCost (MVP는 **Cooldown only** 정책으로 0 고정)
- HitRule
- PushForce (넉백 강도)
- PushDirection (넉백 방향 계산 규칙)

## 5) MVP 운용 정책
- 슬롯 수는 최대 12개까지 지원 가능하도록 문서/구조 설계
- 초기 구현은 **핵심 전투 검증 우선** (스킬 수는 작게 시작 후 확대)
- 초기 스킬 풀 크기: **12개로 확정** (각 태그 균형)

## 6) 밸런스 가드레일(초안)
- 넉백 스킬이 승리 수단으로 유효해야 하나, 대응 수단(방어/이동기)도 반드시 존재해야 함
- 모든 스킬은 카운터 가능성이 있어야 함
- 즉사/무한 CC 조합 금지 (넉백 연계 포함)
- 고성능 스킬일수록 쿨다운/시전 리스크 증가
